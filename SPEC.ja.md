# cronrun 仕様

## 概要

`cronrun` は、指定したコマンドを

-   crontab形式のスケジュール
-   単純なループ実行

のいずれかで実行するコマンドラインツールである。

本ツールは **常駐デーモンではなく、実行中のみスケジュールを管理する**。\
プロセス終了とともにスケジュールも終了する。
利用者向けのインストール方法と実行例は `README.md` / `README.ja.md` に記載する。

------------------------------------------------------------------------

# 実行モード

## 1. cron モード

### 書式

    cronrun "<CRONTAB_LINE>"

### 入力形式

`<CRONTAB_LINE>` は crontab の1行と同じ形式とする。

    <minute> <hour> <day-of-month> <month> <day-of-week> <command> [args...]

例

    cronrun "* * * * * php worker.php"
    cronrun "*/5 * * * * flock -n /tmp/worker.lock php worker.php"
    cronrun "0 2 * * * ./backup.sh"

### 挙動

1.  文字列を **先頭5フィールド**で分割する
2.  前半5フィールドを **cron式**
3.  残りを **実行コマンド**として扱う

例

    */5 * * * * php worker.php

↓

    schedule = */5 * * * *
    command  = php worker.php

### スケジューリング

-   cron式に従って次回実行時刻を計算する
-   次回時刻まで待機する
-   到達したらコマンドを実行する
-   コマンド終了後、次の実行時刻を計算する

### 時刻基準

-   スケジューリングは **ローカルタイム（OSのローカルタイムゾーン）** を基準とする
-   `TZ` 指定など、ツール独自のタイムゾーン指定は提供しない
-   夏時間（DST）切り替え時は、ローカルタイムに従って評価する
-   存在しないローカル時刻（春の繰り上げで欠落する時刻）はスキップされる
-   同じローカル時刻が2回発生する場合（秋の繰り下げ）は、該当時刻ごとに評価されうる

### 実行重複

-   同時実行制御は行わない
-   コマンドが実行中でも次のスケジュールが来た場合は新たに実行する
-   同時実行制御が必要な場合は `flock` などを利用する

------------------------------------------------------------------------

# 2. loop モード

### 書式

    cronrun --loop <command> [args...]

例

    cronrun --loop php worker.php
    cronrun --loop flock -n /tmp/worker.lock php worker.php

### 挙動

以下を繰り返す

    コマンド実行
    終了待ち
    再実行

コマンド終了後は **即座に次の実行を開始する**。

------------------------------------------------------------------------

# シグナル処理

`SIGINT`（Ctrl+C）または `SIGTERM` を受信した場合の挙動は以下とする。

### 共通動作

1.  新しいスケジュールの開始を停止する
2.  現在実行中のコマンドがある場合は **終了を待つ**
3.  コマンド終了後にプロセスを終了する

### 実行中のコマンドが無い場合

-   即座に終了する

### 子プロセスへのシグナル転送

-   親プロセスは子プロセスへ明示的な終了シグナル送信を行わない
-   親プロセスは「新規起動を止める + 既存子プロセスの終了待ち」のみを行う

------------------------------------------------------------------------

# コマンド実行

-   コマンドは **子プロセスとして実行する**
-   標準入力 / 標準出力 / 標準エラーは
    **親プロセスと同じストリームを使用する**
-   コマンド終了コードはログ化や制御に使用しない

### 実行方式

-   `cron` モード:
    -   `<CRONTAB_LINE>` は先頭5フィールドを cron式として切り出し、残りを「コマンド文字列」として扱う
    -   「コマンド文字列」はシェル経由で実行する（`/bin/sh -c` 相当）
-   `loop` モード:
    -   `--loop` に1つだけ引数を渡した場合、その文字列をシェル経由で実行する（`/bin/sh -c` 相当）
        -   例: `cronrun --loop "date; sleep 2"`
    -   `--loop` に複数引数を渡した場合、引数列として受け取った内容を1つのコマンド文字列として組み立て、シェル経由で実行する

------------------------------------------------------------------------

# プロセスライフサイクル

-   本ツールは **常駐サービスではない**
-   ユーザーが起動している間のみ動作する
-   プロセス終了とともにスケジュールは破棄される

------------------------------------------------------------------------

# 依存仕様

-   cron式は **5フィールド形式**とする

    minute hour day-of-month month day-of-week

-   秒フィールドは扱わない

------------------------------------------------------------------------

# エラー処理

以下の場合はエラーとして終了する

-   cron式が5フィールド未満
-   実行コマンドが存在しない
-   cron式が不正

終了コードは `1` とする。

起動後に `SIGINT` / `SIGTERM` で正常停止した場合の終了コードは `0` とする。

------------------------------------------------------------------------

# CLI 補助オプション

-   `--help`: 使い方を表示して終了コード `0` で終了する
-   `--version`: バージョン文字列を表示して終了コード `0` で終了する
-   `--log`: 標準エラー出力へ実行時ログを出力する

------------------------------------------------------------------------

# メッセージ方針

-   エラーメッセージは標準エラー出力に出す
-   文言は英語で統一する
-   フォーマットは `error: <message>` とする

------------------------------------------------------------------------

# ログ出力（`--log`）

`--log` 指定時は標準エラー出力に実行時ログを出力する。

-   タイムスタンプ形式: ローカルタイムゾーンの ISO 8601
-   `cron` モード:
    -   `cron.next`: 次回実行待機に入る直前
    -   `run.start`: 子プロセス起動時
    -   `run.done`: 子プロセス終了時（実行時間と終了コードを含む）
-   `loop` モード:
    -   `run.start`
    -   `run.done`（実行時間と終了コードを含む）
-   `SIGINT` / `SIGTERM` 受信時:
    -   `signal.received`
    -   `shutdown.waiting`（実行中の子プロセスがある場合のみ）
    -   `shutdown.complete`（終了処理が完了した時）

------------------------------------------------------------------------

# 受け入れテスト（最小）

-   `cronrun "* * * * * echo hi"` が1分単位で起動されること
-   `cronrun --loop echo hi` が連続実行されること
-   不正cron式（例: `cronrun "x * * * * echo hi"`）で終了コード `1` になること
-   コマンド未指定（例: `cronrun "* * * * *"`）で終了コード `1` になること
-   実行中に `SIGTERM` を送ると、新規起動を止めて終了待ち後に終了コード `0` で終わること

------------------------------------------------------------------------

# 非対象

本ツールでは以下は扱わない

-   crontab ファイル管理
-   常駐デーモン
-   systemd timer
-   ジョブ履歴管理
-   ログ管理
-   同時実行制御
